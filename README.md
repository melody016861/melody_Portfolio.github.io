## Flask CI/CD 自動化部署專案

本專案展示了一個完整的 **DevOps 自動化流程**，整合 **Flask、Docker、Kubernetes (Kind)** 與 **GitHub Actions**，實現從程式提交到自動部署的全流程架構。  

### 🚀 專案概述
以 Flask 為後端框架，透過 Docker 容器化應用程式，並以 Kind 建立本地 Kubernetes 叢集進行部署。  
GitHub Actions 負責自動化 **建置（Build）→ 測試（Test）→ 部署（Deploy）** 流程，完整模擬雲端部署環境。  

---

### 🧩 架構說明
Developer → GitHub → GitHub Actions (CI/CD) → Docker Image → Kind Cluster (Kubernetes)

| 層級 | 技術 | 功能 |
|------|------|------|
| 應用層 | Flask | Python Web 應用程式 |
| 容器化 | Docker | 打包與封裝應用程式 |
| 叢集管理 | Kubernetes (Kind) | 模擬雲端環境的部署與服務 |
| 自動化 | GitHub Actions | 持續整合與自動部署流程 |

### ⚙️ 技術堆疊
- **Language:** Python 3.x  
- **Framework:** Flask  
- **Containerization:** Docker  
- **Cluster:** Kind (Kubernetes in Docker)  
- **Automation:** GitHub Actions  
- **Version Control:** Git + GitHub  

### 📂 專案結構
<pre><code>```text flask-k8s-cicd/ ├── app.py ├── Dockerfile ├── requirements.txt ├── k8s/ │ ├── deployment.yaml │ └── service.yaml ├── .github/ │ └── workflows/ │ └── deploy.yml └── README.md ```</code></pre>

### 🔁 Pipeline 流程
1. 提交程式碼至 GitHub `main` 分支  
2. 觸發 GitHub Actions 自動建置流程  
3. 進行 CI 階段（Docker Build + 測試）  
4. 進行 CD 階段（部署至 Kind Cluster）  
5. Flask App 自動更新完成  

### 🧾 CI/CD 設定範例
```yaml
name: CI/CD Pipeline

on:
  push:
    branches: [ main ]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v3

      - name: Build Docker image
        run: docker build -t flask-app .

      - name: Run unit tests
        run: pytest || echo "No tests configured"

      - name: Deploy to Kind
        run: |
          kind create cluster --name flask-k8s
          kind load docker-image flask-app
          kubectl apply -f k8s/
```
